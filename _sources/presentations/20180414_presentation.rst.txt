Living in a text world
----------------------


Why text?
^^^^^^^^^

Text has these advantages:

* Edit via a wide variety of open source editors available on command-line-only cloud servers (`Emacs <https://www.gnu.org/software/emacs/>`_ and `Vim <https://www.vim.org/about.php>`_)

* Version control for authoring code, documents, and infrastructure as code (see `The Noflake Manifesto <http://noflake.org/>`_)

  * Text-based markup languages like Markdown, ReStructuredText (see `Comparison of document markup languages <https://en.wikipedia.org/wiki/Comparison_of_document_markup_languages>`_) are the basis for most static websites (offering inexpensive, secure, and fast hosting)


The terminal
^^^^^^^^^^^^

There is a `Terminal emulator <https://en.wikipedia.org/wiki/Terminal_emulator>`_ (which emulates a terminal - see `List of terminal emulators <https://en.wikipedia.org/wiki/List_of_terminal_emulators>`_) and also a `Terminal multiplexer <https://en.wikipedia.org/wiki/Terminal_multiplexer>`_ (which multiplexes several terminal emulators, but is not a terminal emulator itself).

We describe starting a `tmux <https://en.wikipedia.org/wiki/Tmux>`_ session here:

.. code-block:: bash
  :emphasize-lines: 1,5,8-15,18-

  hacker@meetup:~$ cat ~/bin/tmux-meetup.sh
  #!/usr/bin/env bash

  SESSIONNAME="meetup"
  tmux has-session -t $SESSIONNAME &> /dev/null
  if [ $? != 0 ]
   then
      tmux new-session -s $SESSIONNAME -n meetup -d
      tmux send-keys -t $SESSIONNAME 'cd ~/Persistent/meetup/pentest-meetup/' C-m
      tmux splitw -v -p 30 -t 0
      tmux send-keys -t $SESSIONNAME 'cd ~/Persistent/meetup/pentest-meetup/' C-m
      tmux splitw -h -t 1
      tmux send-keys -t $SESSIONNAME 'cd ~/Persistent/meetup/pentest-meetup/' C-m
      tmux send-keys -t $SESSIONNAME 'source bin/activate' C-m
      tmux selectp -t 0
  fi

  tmux attach -t $SESSIONNAME
  hacker@meetup:~$ tmux-meetup.sh

The above opens a ``tmux`` session with a 70/30% vertical split, and the bottom 30% horizontally split in 2. Entering :kbd:`C-b d` detaches the session and after logging off and on again you can re-attach via:

.. code-block:: bash
  :emphasize-lines: 1-2,4

  tmux ls           # show session still running after logout
  tmux a -t meetup  # reattach where you left off
  # Or using tmux-meetup.sh which checks for existing session
  tmux-meetup.sh

So you can open up a session, detach, logoff, login from another host, then pick up where you left off.

For introductions to ``tmux`` try `How to Use tmux the Terminal Multiplexer <https://linode.com/docs/networking/ssh/persistent-terminal-sessions-with-tmux/>`_ or archlinux - tmux <https://wiki.archlinux.org/index.php/tmux>`_. For cheatsheets try `tmux shortcuts & cheatsheet <https://gist.github.com/MohamedAlaa/2961058>`_ or `tmux cheat sheet <https://gist.github.com/andreyvit/2921703>`_.


A minimal, secure setup
^^^^^^^^^^^^^^^^^^^^^^^

Perhaps the most minimal, secure setup is using a Chromebook/Chromebox to connect to a cloud server using Chrome's Secure Shell app (SSH). From there use  ``tmux``, ``emacs-nox``/``vim``, `gnuplot <http://www.gnuplot.info/>`_ (command line graphing), and other command line tools to author this web site. The Chromebook Chrome browser can be used to view the test website and download files (to upload to Google Cloud Storage where ``gsutil cp gs://...`` can copy it to the cloud server).

Many users work primarily from the command line:

* Email and contacts - `Fast CLI Tools: gmail <https://blog.afoolishmanifesto.com/posts/fast-cli-tools-and-gmail/>`_

* ``gnuplot`` - `gnuplot is Super Handy <https://blog.afoolishmanifesto.com/posts/gnuplot-super-handy/>`_

* Automated setup of `dotfiles <https://github.com/frioux/dotfiles>`_ (configuration) - `My Mobile (shell) Home <https://blog.afoolishmanifesto.com/posts/my-mobile-shell-home/>`_

* Tons of emacs packages, like `Org Mode <https://orgmode.org/>`_ (notes, TODO lists, ...)
