11.14.2.1.2. Getting Docker¶

11.14.2.1.2.1. Installing docker in Linux¶

For Debian/Kali follow Get Docker CE for Debian and for Ubuntu follow Get Docker CE for Ubuntu. Kali will fail the add-apt-repository so manually add deb [arch=amd64] https://download.docker.com/linux/debian buster stable. Ubuntu bionic (18.04) not yet supported so temporarity use Ubuntu artful (17.10) repo.

From Post-installation steps for Linux make sure to at least sudo usermod -aG docker $USER to insure your account can run docker without sudo.

11.14.2.1.2.2. CoreOS in Linode¶

See linode Guides & Tutorial >> Applications >> Containers for container articles, with Use CoreOS Container Linux on Linode for specific differences deploying CoreOS on Linode:

  • User is core and not root

  • Does not use grub2 to boot, instead using “Direct Disk”

  • Not compatible with Linode kernels

  • swap file cannot be present in /etc/fstab

  • The CoreOS Update Philosophy

    … automates software updates to ensure better security and reliability of machines and containers running in large-scale clusters. Operating system updates and security patches are regularly pushed to CoreOS Container Linux machines without requiring intervention by administrators. When applications are distributed, these automatic updates dramatically improve security without causing service downtime.

  • Reboot strategies on updates are one of

    Reboot strategies on update¶
    STRATEGY DESCRIPTION
    etcd-lock Reboot after first taking a distributed lock in etcd
    reboot Reboot immediately after an update is applied
    off Do not reboot after updates are applied

11.14.2.1.3. A simple ubuntu shell¶

Start with Get Started, Part 1: Orientation and setup then Get Started, Part 2: Containers for a simple start. After that try:

docker --version
docker version
docker info

docker run hello-world

docker run -it ubuntu bash
# In the bash shell:
cat /etc/os-release  # container OS ubuntu bionic
uname -a  # kernel of docker host
hostname  # = docker container id
ps -ef  # running only bash + ps
cat /proc/meminfo | grep MemTotal  # same memory as docker host
ls -a /dev  # many missing devices including disks and network
ls /boot  # empty
# missing commands
lsb_release -cs  # no lsb_release command
ip link show  # no ip command
ss -tnlp  # no ss command
systemctl  # no systemctl command
cat /etc/fstab  # UNCONFIGURED FSTAB FOR BASE SYSTEM
# modify the filesystem
echo hello > /hello
exit

# To run the same container and see the modifications
# "docker run -it ubuntu bash" would start a new container
docker start -ai $(docker ps -lq)
ls -al /
exit

# To persist the changes into an image
docker commit $(docker ps -lq) my_new_image
docker images
docker run -it my_new_image bash
ls -al /
exit

# Clean up
docker stop $(docker ps -aq)  # stop all running containters
docker rm $(docker ps -aq)  # remove all containers
# docker rmi $(docker images -q)
docker image rm -f my_new_image
