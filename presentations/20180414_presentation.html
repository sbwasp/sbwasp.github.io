11.1.2.1. Living in a text world¶

11.1.2.1.1. Why text?¶

Text has these advantages:

  • Edit via a wide variety of open source editors available on command-line-only cloud servers (Emacs and Vim)
  • Version control for authoring code, documents, and infrastructure as code (see The Noflake Manifesto)
    • Text-based markup languages like Markdown, ReStructuredText (see Comparison of document markup languages) are the basis for most static websites (offering inexpensive, secure, and fast hosting)

11.1.2.1.2. The terminal¶

There is a Terminal emulator (which emulates a terminal - see List of terminal emulators) and also a Terminal multiplexer (which multiplexes several terminal emulators, but is not a terminal emulator itself).

We describe starting a tmux session here:

hacker@meetup:~$ cat ~/bin/tmux-meetup.sh
#!/usr/bin/env bash

SESSIONNAME="meetup"
tmux has-session -t $SESSIONNAME &> /dev/null
if [ $? != 0 ]
 then
    tmux new-session -s $SESSIONNAME -n meetup -d
    tmux send-keys -t $SESSIONNAME 'cd ~/Persistent/meetup/pentest-meetup/' C-m
    tmux splitw -v -p 30 -t 0
    tmux send-keys -t $SESSIONNAME 'cd ~/Persistent/meetup/pentest-meetup/' C-m
    tmux splitw -h -t 1
    tmux send-keys -t $SESSIONNAME 'cd ~/Persistent/meetup/pentest-meetup/' C-m
    tmux send-keys -t $SESSIONNAME 'source bin/activate' C-m
    tmux selectp -t 0
fi

tmux attach -t $SESSIONNAME
hacker@meetup:~$ tmux-meetup.sh

The above opens a tmux session with a 70/30% vertical split, and the bottom 30% horizontally split in 2. Entering C-b d detaches the session and after logging off and on again you can re-attach via:

tmux ls           # show session still running after logout
tmux a -t meetup  # reattach where you left off
# Or using tmux-meetup.sh which checks for existing session
tmux-meetup.sh

So you can open up a session, detach, logoff, login from another host, then pick up where you left off.

For introductions to tmux try How to Use tmux the Terminal Multiplexer or archlinux - tmux <https://wiki.archlinux.org/index.php/tmux>`_. For cheatsheets try tmux shortcuts & cheatsheet or tmux cheat sheet.

11.1.2.1.3. A minimal, secure setup¶

Perhaps the most minimal, secure setup is using a Chromebook/Chromebox to connect to a cloud server using Chrome’s Secure Shell app (SSH). From there use tmux, emacs-nox/vim, gnuplot (command line graphing), and other command line tools to author this web site. The Chromebook Chrome browser can be used to view the test website and download files (to upload to Google Cloud Storage where gsutil cp gs://... can copy it to the cloud server).

Many users work primarily from the command line:

  • Email and contacts - Fast CLI Tools: gmail
  • gnuplot - gnuplot is Super Handy
  • Automated setup of dotfiles (configuration) - My Mobile (shell) Home
  • Tons of emacs packages, like Org Mode (notes, TODO lists, …)
